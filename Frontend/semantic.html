Semantic vs Non-Semantic HTML Elements

Semantic HTML

Semantic elements describe their meaning in a way that both humans and machines can understand. They provide better structure and accessibility for the content.

Example:

<header>
    <h1>Welcome to My Website</h1>
    <nav>
        <ul>
            <li><a href="#home">Home</a></li>
            <li><a href="#about">About</a></li>
            <li><a href="#contact">Contact</a></li>
        </ul>
    </nav>
</header>

<main>
    <section>
        <h2>About Us</h2>
        <p>This section describes our company.</p>
    </section>
    <article>
        <h2>Latest News</h2>
        <p>Here are the latest updates from our company...</p>
    </article>
</main>

<footer>
    <p>© 2024 My Website. All rights reserved.</p>
</footer>

Non-Semantic HTML

Non-semantic elements do not define the content they hold. These tags are used for layout purposes without providing meaning to the content.

Example:

<div>
    <h1>Welcome to My Website</h1>
    <div>
        <a href="#home">Home</a> | <a href="#about">About</a> | <a href="#contact">Contact</a>
    </div>
</div>

<div>
    <h2>About Us</h2>
    <p>This section describes our company.</p>
</div>

<div>
    <h2>Latest News</h2>
    <p>Here are the latest updates from our company...</p>
</div>

<div>
    <footer>
        <p>© 2024 My Website. All rights reserved.</p>
    </footer>
</div>

Comparison Between Semantic and Non-Semantic Elements

Semantic Elements Non-Semantic Elements
Meaning: Clearly defines the content inside it, describing its purpose. Meaning: Does not define or describe its content. Used for structure and layout.
Examples: <header>, <footer>, <article>, <section>, <nav> Examples: <div>, <span>, <b>, <i>, <u>
Accessibility: Improves accessibility for screen readers and search engines (SEO-friendly). Accessibility: Does not provide any meaning for screen readers or search engines.
Readability: Improves code readability and helps developers understand the content structure. Readability: May make the code harder to understand since it lacks descriptive meaning.
Layout Purpose: Primarily used to structure and describe content. Layout Purpose: Primarily used for styling and layout.
